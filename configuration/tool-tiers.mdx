Cruncher exposes its calculator functions in three tiers, each a strict superset of the previous one. The active tier is controlled by the `CRUNCHER_TOOL_SET` environment variable. Choosing the right tier lets you trade a smaller context window footprint for a narrower set of registered MCP tools â€” or open up the full suite when your workflow demands it.

## Tier overview

| Tier | Tools | Token budget | Best for |
|------|-------|-------------|----------|
| `minimal` | 5 | ~160 tokens | Lightweight agents, simple arithmetic |
| `standard` | 34 | ~1,150 tokens | General-purpose AI assistant math (default) |
| `full` | 43 | ~1,500 tokens | Power users, multi-step and memory workflows |

<Note>
  Even in `minimal` mode, `evaluate_expression` handles complex math including trigonometry, logarithms, and constants â€” individual tools like `sine` and `sqrt` simply are not registered as separate MCP tool calls. This can save up to 90% on context tokens compared to the `full` tier.
</Note>

## Minimal tier (5 tools)

The minimal tier registers only the five core arithmetic tools. All complex math must go through `evaluate_expression`, which supports trig functions, logarithms, roots, and all named constants inline.

| # | Tool | Description |
|---|------|-------------|
| 1 | `evaluate_expression` | Evaluate a plain-text math expression string |
| 2 | `add` | Addition (a + b) |
| 3 | `subtract` | Subtraction (a - b) |
| 4 | `multiply` | Multiplication (a Ă— b) |
| 5 | `divide` | Division (a Ă· b) |

**When to use minimal:** When token budget is the primary constraint, when the AI primarily uses `evaluate_expression` for everything, or when you are building a lightweight calculator agent that does not need typed statistical or unit-conversion tools.

The full tier adds nine more tools on top of the standard set: memory functions, batch execution, result caching, base conversion, and percentile calculation.

| # | Tool | Category |
|---|------|----------|

**When to use full:** When you need persistent memory across turns, multi-step batch calculations, base conversion (binary, octal, hex), or percentile analysis. Expect ~1,500 tokens of context overhead.

## Configuring the tier

Set `CRUNCHER_TOOL_SET` in the `env` block of your MCP server configuration. Restart the MCP client after changing the value.

<CodeGroup>

```json Minimal tier
{
  "mcpServers": {
    "cruncher": {
      "command": "npx",
      "args": ["@slbdn/cruncher-mcp"],
      "env": {
        "CRUNCHER_TOOL_SET": "minimal"
      }
    }
  }
}
```

```json Standard tier (default)
{
  "mcpServers": {
    "cruncher": {
      "command": "npx",
      "args": ["@slbdn/cruncher-mcp"],
      "env": {
        "CRUNCHER_TOOL_SET": "standard"
      }
    }
  }
}
```

```json Full tier
{
  "mcpServers": {
    "cruncher": {
      "command": "npx",
      "args": ["@slbdn/cruncher-mcp"],
      "env": {
        "CRUNCHER_TOOL_SET": "full"
      }
    }
  }
}
```

</CodeGroup>

<Tip>
  If the AI client reports that a tool is not found (for example, `memory_recall` or `percentile`), the server is likely running in `standard` or `minimal` mode. Switch to `CRUNCHER_TOOL_SET=full` and restart your MCP client.
</Tip>
