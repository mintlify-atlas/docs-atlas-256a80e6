Cruncher includes three tools for exponentiation, roots, and number theory. All three are part of the **standard tier** and are available in any configuration except `minimal`. Each call runs in an isolated Node.js worker thread with a timeout (default 3 seconds, configurable via `CRUNCHER_TIMEOUT`) so that a large factorial or a deeply recursive expression can never block the MCP server.

## power

Raises a base to an exponent and returns the result.

<ParamField body="base" type="number" required>
  The base number. Accepts any real number, including negative values and decimals.
</ParamField>

<ParamField body="exponent" type="number" required>
  The exponent. Accepts any real number, including negative values (for reciprocals) and fractional values (for roots).
</ParamField>

<ResponseField name="result" type="number">
  The value of `base ^ exponent`.
</ResponseField>

### Examples

<CodeGroup>

```json Integer exponent
{
  "tool": "power",
  "arguments": { "base": 2, "exponent": 10 }
}
```

```json Fractional exponent (cube root)
{
  "tool": "power",
  "arguments": { "base": 27, "exponent": 0.3333333333333333 }
}
```

```json Negative exponent (reciprocal)
{
  "tool": "power",
  "arguments": { "base": 2, "exponent": -3 }
}
```

```json Decimal base
{
  "tool": "power",
  "arguments": { "base": 1.5, "exponent": 4 }
}
```

</CodeGroup>

```json
{ "result": 1024 }
{ "result": 3 }
{ "result": 0.125 }
{ "result": 5.0625 }
```

<Tip>
For arbitrary roots, use a fractional exponent: the nth root of x is `x ^ (1/n)`. For example, the cube root of 27 is `power(27, 1/3)`. You can also write the full expression in `evaluate_expression` as `27 ^ (1/3)`.
</Tip>

## sqrt

Calculates the square root of a non-negative number.

<ParamField body="value" type="number" required>
  The radicand. Must be greater than or equal to zero. Passing a negative value returns a domain error.
</ParamField>

<ResponseField name="result" type="number">
  The positive square root of `value`.
</ResponseField>

### Examples

<CodeGroup>

```json Perfect square
{
  "tool": "sqrt",
  "arguments": { "value": 144 }
}
```

```json Non-perfect square
{
  "tool": "sqrt",
  "arguments": { "value": 2 }
}
```

```json Zero
{
  "tool": "sqrt",
  "arguments": { "value": 0 }
}
```

</CodeGroup>

```json
{ "result": 12 }
{ "result": 1.4142135623730951 }
{ "result": 0 }
```

### Error case

```json
{
  "tool": "sqrt",
  "arguments": { "value": -9 }
}
```

```json
{
  "error": {
    "code": -32000,
    "message": "Check for: sqrt(negative), log(negative/zero), asin/acos out of [-1,1]"
  }
}
```

<Warning>
`sqrt` does not return complex numbers. Any negative input is a domain error.
</Warning>

## factorial

Calculates the factorial of a non-negative integer (n!).

<ParamField body="n" type="number" required>
  A non-negative integer. Decimal values and negative numbers are not accepted and will produce a validation error. There is no enforced upper bound in the schema, but very large values (e.g., `n > 100,000`) will be terminated by the worker thread timeout.
</ParamField>

<ResponseField name="result" type="number">
  The value of n! — the product of all positive integers from 1 to n. By definition, `0! = 1`.
</ResponseField>

### Examples

<CodeGroup>

```json Small factorial
{
  "tool": "factorial",
  "arguments": { "n": 5 }
}
```

```json Larger factorial
{
  "tool": "factorial",
  "arguments": { "n": 10 }
}
```

```json Zero factorial
{
  "tool": "factorial",
  "arguments": { "n": 0 }
}
```

```json Typical use case
{
  "tool": "factorial",
  "arguments": { "n": 15 }
}
```

</CodeGroup>

```json
{ "result": 120 }
{ "result": 3628800 }
{ "result": 1 }
{ "result": 1307674368000 }
```

### Timeout protection

Factorial grows extremely fast. Computing `factorial(1000000)` would run for far longer than the 3-second default timeout. When the worker thread is terminated for exceeding the limit, Cruncher returns:

```json
{
  "error": {
    "code": -32000,
    "message": "Execution timed out after 3000ms."
  }
}
```

<Note>
On a timeout error, do not retry with the same input. If you need a higher limit, increase `CRUNCHER_TIMEOUT` (in milliseconds) in your MCP server configuration — for example, set it to `10000` for a 10-second window. However, very large factorials also produce numbers that exceed JavaScript's safe integer range, which may cause unexpected precision loss.
</Note>

### Validation errors

Passing a decimal or a negative number returns a `-32602` validation error before any computation runs:

```json
{
  "tool": "factorial",
  "arguments": { "n": 5.5 }
}
```

```json
{
  "error": {
    "code": -32602,
    "message": "Validation Error: 'n' must be a non-negative integer."
  }
}
```
