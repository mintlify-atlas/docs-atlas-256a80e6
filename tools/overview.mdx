Cruncher exposes up to 43 MCP tools depending on the active tier. This page gives you a complete picture of what is available, how tools are grouped, and what to expect when something goes wrong. Use the category cards below to jump to detailed documentation for each group.

## Tier system

Cruncher controls which tools are registered with the MCP client via the `CRUNCHER_TOOL_SET` environment variable. Each tier is a strict superset of the one before it.

| Tier | Tools registered | Token budget | Best for |
|------|-----------------|--------------|----------|
| `minimal` | 5 | ~160 tokens | Lightweight agents; all complex math via `evaluate_expression` |
| `standard` *(default)* | 34 | ~1,150 tokens | General-purpose AI assistants: arithmetic, trig, stats, constants, unit conversion |
| `full` | 43 | ~1,500 tokens | Power users: adds memory, batch, cache, base conversion, percentile |

<Note>
Even in `minimal` mode, `evaluate_expression` handles complex math internally. Individual tools like `sine` and `sqrt` simply are not registered as separate MCP calls, saving up to 90% of context tokens.
</Note>

For configuration instructions and environment variable reference, see [Tool Tiers](/configuration/tool-tiers).

## Validation errors (JSON-RPC -32602)

Cruncher validates every tool call against a JSON Schema before execution. If the schema check fails, it returns a JSON-RPC error with code `-32602` and a message that includes the exact property path that failed.

Common causes:

- **Wrong type** — passing a string where a number is required (e.g., `"5"` instead of `5`)
- **Invalid enum value** — using `"deg"` instead of `"degrees"`, or `"speed_of_light"` instead of `"c"`
- **Wrong shape** — passing `"1,2,3"` instead of `[1, 2, 3]` for an array argument

```json
{
  "error": {
    "code": -32602,
    "message": "Validation Error: 'a' must be of type number, got string."
  }
}
```

<Warning>
Always send numeric arguments as JSON numbers, not strings. Enum values must match exactly — Cruncher does not normalize casing for argument values.
</Warning>

## Fuzzy tool name matching

If the AI client calls a tool name that does not exist, Cruncher uses Levenshtein distance plus prefix matching to suggest the closest valid tool instead of returning a generic error.

| Typo called | Suggestion returned |
|-------------|-------------------|
| `fact` | `factorial` |
| `sinn` | `sine` |
| `squrt` | `sqrt` |
| `adddd` | `add` |
| `divid` | `divide` |
| `totally_wrong` | *(no suggestion — too different)* |

```json
{
  "error": {
    "code": -32601,
    "message": "Tool 'sinn' not found. Did you mean 'sine'?"
  }
}
```
