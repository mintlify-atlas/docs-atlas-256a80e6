Cruncher's `convert_unit` tool converts between 80+ units across 8 categories — length, weight, temperature, area, volume, time, speed, and digital storage. It is available in the standard tier (the default). All unit identifiers are case-insensitive, so `"KM"`, `"Km"`, and `"km"` are all accepted.

<Tip>
  Always use `convert_unit` rather than asking your AI to calculate the conversion factor manually. AI models frequently hallucinate conversion ratios, especially for uncommon units.
</Tip>

---

## convert_unit

<ParamField body="value" type="number" required>
  The numeric quantity to convert.
</ParamField>

<ParamField body="category" type="string" required>
  The unit category. Must be one of: `length`, `weight`, `temperature`, `area`, `volume`, `time`, `speed`, `digital_storage`.
</ParamField>

<ParamField body="from" type="string" required>
  The source unit identifier (e.g., `"mi"`, `"kg"`, `"c"`). Case-insensitive. See the unit tables below for all valid identifiers.
</ParamField>

<ParamField body="to" type="string" required>
  The target unit identifier. Must be in the same category as `from`.
</ParamField>

### Response format

<ResponseField name="value" type="number">
  The original input value.
</ResponseField>

<ResponseField name="from" type="string">
  The source unit identifier.
</ResponseField>

<ResponseField name="to" type="string">
  The target unit identifier.
</ResponseField>

<ResponseField name="result" type="number">
  The converted value in the target unit.
</ResponseField>

<ResponseField name="category" type="string">
  The unit category (echoes back the `category` argument).
</ResponseField>

<Warning>
  Temperature conversions are **non-linear** and use the proper formulas — not simple multiplication by a constant factor:

  - **°C → °F:** `(°C × 9/5) + 32`
  - **°F → °C:** `(°F − 32) × 5/9`
  - **°C → K:** `°C + 273.15`
  - **K → °C:** `K − 273.15`

</Warning>

## Notes

**Case-insensitivity:** Unit identifiers are matched case-insensitively. `"KM"`, `"Km"`, and `"km"` all resolve to kilometer. The only exception is `digital_storage`, where byte (`B`) and bit (`b`) differ by case — in that category, the original case is preserved for disambiguation.

**Temperature:** Always use `convert_unit` for temperature. It applies the correct non-linear formulas. A linear scaling factor will give wrong results for Celsius ↔ Fahrenheit and Kelvin conversions.
