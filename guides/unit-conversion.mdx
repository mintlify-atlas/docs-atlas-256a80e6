---

Cruncher's `convert_unit` tool handles over 80 unit conversions across 8 measurement categories, including temperature conversions that require non-linear formulas. You pass a numeric value, the category it belongs to, the unit you are converting from, and the unit you want to convert to. Cruncher returns a structured object with the original value, both unit names, the converted result, and the category.

## Tool signature

```json
{
  "tool": "convert_unit",
  "arguments": {
    "value": 5,
    "category": "length",
    "from": "mi",
    "to": "km"
  }
}
```

| Argument | Type | Description |
|----------|------|-------------|
| `value` | number | The numeric quantity to convert |
| `category` | string | One of the 8 supported category names |
| `from` | string | Unit to convert from (case-insensitive) |
| `to` | string | Unit to convert to (case-insensitive) |

<Note>
  Unit identifiers are case-insensitive. `"KM"`, `"Km"`, and `"km"` all refer to the same unit.
</Note>

## Response format

Every successful call returns an object with five fields:

```json
{
  "value": 5,
  "from": "mi",
  "to": "km",
  "result": 8.04672,
  "category": "length"
}
```

| Field | Description |
|-------|-------------|
| `value` | The original input value |
| `from` | The source unit as provided |
| `to` | The target unit as provided |
| `result` | The converted numeric result |
| `category` | The category name |

|-----------|------------|

| Nanometer | `nm` |
| Mile | `mi` |
| Yard | `yd` |
| Foot | `ft` |
| Inch | `in` |
| Nautical mile | `nmi` |

### Weight

| Unit name | Identifier |
|-----------|------------|

| Gram | `g` |
| Milligram | `mg` |
| Microgram | `ug` |
| Pound | `lb` |
| Ounce | `oz` |
| Stone | `st` |
| Metric ton | `t` |

### Temperature

| Unit name | Identifier |
|-----------|------------|
| Celsius | `c` |
| Fahrenheit | `f` |
| Kelvin | `k` |

<Warning>
  Temperature conversions use non-linear formulas — they are not simple multiplication by a fixed factor. Always use `convert_unit` for temperature; do not attempt to convert manually or via `evaluate_expression`.
</Warning>

<CodeGroup>

```json Celsius to Fahrenheit
{
  "value": 100,
  "category": "temperature",
  "from": "c",
  "to": "f"
}
```

```json Fahrenheit to Kelvin
{
  "value": 32,
  "category": "temperature",
  "from": "f",
  "to": "k"
}
```

```json Kelvin to Celsius
{
  "value": 273.15,
  "category": "temperature",
  "from": "k",
  "to": "c"
}
```

</CodeGroup>

## Availability

`convert_unit` is available in the **standard** tier (the default). You do not need to set `CRUNCHER_TOOL_SET=full` to use it. See [tool tiers](/configuration/tool-tiers) for the full breakdown.
