Cruncher is a scientific calculator built for the Model Context Protocol (MCP). It gives any MCP-compatible AI client — Claude Desktop, Cursor, Cline, Goose, and others — access to 43 mathematical tools spanning arithmetic, trigonometry, statistics, unit conversion, and physical constants. Everything runs in pure Node.js with no external dependencies, worker-thread isolation for safety, and integer-scaling math for decimal accuracy.

## Why Cruncher?

AI language models hallucinate on math. Cruncher eliminates that by offloading every calculation to a validated, deterministic tool — no guessing, no floating-point drift, no infinite loops.

<CardGroup cols={2}>
  <Card title="Zero dependencies" icon="box">
    Pure Node.js — no mathjs, no zod, no SDK. A single `cruncher.js` file is all you need.
  </Card>
  <Card title="Accurate decimal math" icon="calculator">
    Integer-scaling arithmetic ensures `0.1 + 0.2 === 0.3`. No IEEE 754 drift.
  </Card>
  <Card title="Worker thread isolation" icon="shield">
    Heavy calculations run in isolated threads with a configurable timeout. The server never freezes.
  </Card>
  <Card title="Tiered tool exposure" icon="layer-group">
    Choose minimal (5 tools), standard (34 tools), or full (43 tools) to control context token usage.
  </Card>
</CardGroup>

## Get started in minutes

<Steps>
  <Step title="Run Cruncher via npx">
    No installation needed. Add this to your AI client's MCP configuration:

    ```json claude_desktop_config.json
    {
      "mcpServers": {
        "cruncher": {
          "command": "npx",
          "args": ["@slbdn/cruncher-mcp"]
        }
      }
    }
    ```
  </Step>
  <Step title="Restart your AI client">
    Quit and relaunch your AI application so it picks up the new MCP server configuration.
  </Step>
  <Step title="Start calculating">
    Ask your AI assistant to use Cruncher for any math. It will automatically call the right tool — no special prompting required.

    > "Calculate the standard deviation of [85, 92, 78, 90, 88]"
    >
    > "Convert 26.2 miles to kilometers"
    >
    > "Evaluate: sin(pi/4) + sqrt(16) + log10(100)"
  </Step>
</Steps>
